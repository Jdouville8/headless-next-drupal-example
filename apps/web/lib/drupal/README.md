# `lib/drupal` — contract for Agents C & D

This directory is the only place that knows about Drupal. Components must never
import from here directly; they consume the flat `Article` type from
`@/types/article`.

## The contract

`@/types/article` exports the full type surface:

```ts
import type {
  Article,
  ArticleAttributes,
  Breadcrumb,
  Tag,
  HeroMedia,
  BodyBlock,
  RichTextBlock,
  PullQuoteBlock,
  PressContact,
  ShareTarget,
} from "@/types/article";
```

This file is generated by hand from `JSON_CONTRACT.md` at the repo root and is
the **single source of truth** for both sides of the wire. If you need a field
the type doesn't expose, update `JSON_CONTRACT.md` first, then this file, then
the adapter — in that order.

## Component-level development

For local dev and Storybook-style isolation, import the canonical mock:

```ts
import { mockArticle } from "@/lib/mock/article";
```

The mock is strictly typed against `Article`, so if it compiles, your component
is consuming a real-world shape.

## Mock vs. live toggle

`getArticleBySlug(slug)` short-circuits to `mockArticle` when
`NEXT_PUBLIC_USE_MOCK_DATA=true` (the default). With the flag off, the adapter
hits Drupal's JSON:API and normalizes the envelope into the flat `Article`
shape via `normalizeArticle`.

The two paths return the **same shape**, so component code should be identical
in either mode.

## `data-jsonblock` attributes are mandatory

Every component root that renders content from the Article must carry a
`data-jsonblock` attribute identifying the field it owns. The reference panel
(`components/dev/ArticleReferencePanel`) uses these to highlight regions and
show the matching JSON path. Suggested values:

| Component       | `data-jsonblock`                  |
|-----------------|-----------------------------------|
| `SiteHeader`    | `site.header`                     |
| `Breadcrumb`    | `attributes.breadcrumb`           |
| `HeroMedia`     | `hero`                            |
| `ArticleHero`   | `attributes.title+summary+meta`   |
| `RichText`      | `body[i].html`                    |
| `PullQuote`     | `body[i]`                         |
| `ContactBlock`  | `contact`                         |
| `LegalNotes`    | `legal`                           |
| `TagList`       | `attributes.tags`                 |
| `SiteFooter`    | `site.footer`                     |

Use the index of the block in the `body` array for `[i]`.

## Revalidation

Drupal posts to `POST /api/revalidate?secret=…&path=/news/<slug>&tag=article`.
The shared secret comes from `REVALIDATE_SECRET`. The route uses
`revalidatePath` for direct invalidation and `revalidateTag('article')` so any
fetch tagged in `client.ts` is busted in one shot.
